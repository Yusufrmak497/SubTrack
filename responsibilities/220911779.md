## Technical Decisions I Took
1. **PostgreSQL Migration:** Chose PostgreSQL over SQLite to support production-level relational features and better concurrency for multi-client access (Web + Extension).
2. **Rate Limiting:** Implemented a default 60 req/min limit to protect the API from automated abuse, while allowing sufficient overhead for human usage.
3. **M:N Relationship:** Implemented a junction table for Tags to allow flexible, many-to-many subscription labels — a level above standard 1:N categorisation.
4. **Async FX Integration:** Chose `httpx` for non-blocking external API calls, ensuring the system remains responsive even if the currency provider is slow.
5. **iCalendar Standard:** Used the `.ics` standard to ensure compatibility.
6. **High-Performance Interaction Design:** Leveraged GSAP for hardware-accelerated motion orchestrations and implemented a glassmorphic design system to ensure the application meets modern aesthetic standards and provides fluid, interruptible feedback loops.
